Why cortex

Built around how
Claude actually works.

Context windows are finite and tokens cost money. Cortex makes Claude precise about what it loads, pulling only the relevant slice instead of the whole file.

Search, not load

Claude searches your knowledge base for what it needs instead of loading everything upfront. Synonym expansion finds related terms automatically: search "throttling" and it also finds "rate limit" and "429".

Get started

One command.

Sets up your knowledge base, configures the MCP server, registers context hooks, and wires up Claude Code and VS Code automatically.

$ npx @alaarab/cortex init
Creates ~/.cortex with starter templates, registers your machine, and configures the MCP server. Then restart Claude Code.
$ npx @alaarab/cortex init
Detects VS Code automatically and writes mcp.json for you. Or add it manually: 
{
  "servers": {
    "cortex": {
      "command": "npx",
      "args": ["@alaarab/cortex", "~/.cortex"]
    }
  }
}
Skills

Four skills.
Yours to extend.

Hooks handle saving and context automatically. These four skills cover the things that need a human trigger. Your own workflow skills go in ~/.cortex/global/skills/ and get symlinked into every session.

/cortex-sync
Pull cortex to a new machine or push config changes back. Run this on first setup or after switching machines.
/cortex-init
Scaffold a new project entry with summary, CLAUDE.md, backlog, and skills directory.
/cortex-discover
Research what's missing in a project. Returns a prioritized list of gaps and opportunities.
/cortex-consolidate
Find patterns across all your project LEARNINGS.md files. Surfaces what applies everywhere.
bring your own
Workflow skills like /humanize, /swarm, /pipeline are personal. Drop them in ~/.cortex/global/skills/ and link.sh wires them up. The starter template (bundled in the package) has examples.
