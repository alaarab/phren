Under phren's hood

Three hooks. Zero effort from you.

SessionStart once, on open
git pull --rebase sync ~/.phren from remote
detectProject(cwd) match working directory to a phren project
inject(CLAUDE.md, summary.md) project instructions enter the context window
↳ your agent knows the project before you type anything
UserPromptSubmit every message you send
extractKeywords(prompt) strip stopwords, build bigrams
fts5.search(keywords) synonym expansion, 20-term cap, sub-ms lookup
trustFilter(results) confidence decay, citation check, policy threshold
inject(top3, budget=550) best matches enter context. nothing if nothing matches.
↳ ◆ phren · 3 results in your status line, that's this hook reporting what it found
Stop after every response, including subagents
git add -A ~/.phren stage whatever changed
git commit no-op if nothing changed
background sync worker push later, rebase on conflict, auto-retry
↳ next session, any machine: SessionStart pulls it back down

Phren's tricks

55 MCP tools built in.
Skills for the fancy stuff.

Phren handles the daily work through MCP tools -- search, capture, review, sync. Skills are shortcuts for bigger jobs. Totally optional. Teach him new ones in ~/.phren/global/skills/.

/phren-sync
Move phren to a new machine or push his brain back to the cloud.
/phren-init
Introduce phren to a new project. He'll set up the files and start learning.
/phren-discover
Phren checks himself for gaps -- missing files, stale memories, stuck tasks.
/phren-consolidate
Phren tidies up. Finds patterns, deduplicates, and promotes what matters most.
/phren-profiles
Tell phren which machines see which projects. He keeps the wiring tidy.
teach him yours
Drop skills into ~/.phren/global/skills/ and phren picks them up on the next init. He's a fast learner.
