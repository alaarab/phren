Under the hood

Three hooks. Every session.

SessionStart once, on open
git pull --rebase sync ~/.cortex from remote
detectProject(cwd) match working directory to a cortex project
inject(CLAUDE.md, summary.md) project instructions enter the context window
↳ your agent knows the project before you type anything
UserPromptSubmit every message you send
extractKeywords(prompt) strip stopwords, build bigrams
fts5.search(keywords) synonym expansion, 20-term cap, sub-ms lookup
trustFilter(results) confidence decay, citation check, policy threshold
inject(top3, budget=550) best matches enter context. nothing if nothing matches.
↳ ◆ cortex · 3 results in your status line, that's this hook reporting what it found
Stop after every response, including subagents
git add -A ~/.cortex stage whatever changed
git commit no-op if nothing changed
background sync worker push later, rebase on conflict, auto-retry
↳ next session, any machine: SessionStart pulls it back down
Why cortex

Private. Local.
Git-backed.

No vendor lock-in, no cloud account. Everything lives in a private git repo you own. Profiles keep work and personal separate. What one session learns, the next one gets -- across machines and tools. Stale findings decay. Useful patterns stick.

Lean context. Lower cost.

CLAUDE.md loads the whole file every time. Cortex searches what you actually wrote and injects only what matches: about 550 tokens by default instead of everything. Less token spend means more agents in parallel and cleaner context. Adjust the target with CORTEX_CONTEXT_TOKEN_BUDGET.

Install

One command. Done.

Creates ~/.cortex, registers the MCP server, and wires hooks into Claude Code, Copilot, Cursor, or Codex. Then add a project with cortex init --from-existing ~/project.

$ npx @alaarab/cortex init
Creates ~/.cortex with starter templates, registers your machine, configures MCP, and wires up hooks. Restart your agent.
$ npx @alaarab/cortex init
Detects VS Code automatically and writes mcp.json for you. Or add it manually: 
{
  "servers": {
    "cortex": {
      "command": "npx",
      "args": ["@alaarab/cortex", "~/.cortex"]
    }
  }
}
$ npx @alaarab/cortex init
Detects Cursor automatically and writes hooks to ~/.cursor/hooks.json. Prompt injection and auto-save both work. Note: Cursor has no session-start hook, so context re-injection after compaction isn't available.
For MCP, add to your Cursor settings: 
{
  "mcpServers": {
    "cortex": {
      "command": "npx",
      "args": ["@alaarab/cortex", "~/.cortex"]
    }
  }
}
$ npx @alaarab/cortex init
Detects Codex automatically and writes hooks to codex.json in your cortex directory. Per-project CLAUDE.md is also symlinked as AGENTS.md in each project for full instruction file support.
Context injection, auto-save, and session start/end hooks all work with Codex out of the box.
Workflow

MCP tools handle the work.
Skills are optional shortcuts.

Cortex exposes MCP tools organized by workflow: search and retrieve context, capture findings and backlog, review memory quality, and sync across machines. Skills are shortcuts for common multi-step jobs -- not required for daily use. Add your own in ~/.cortex/global/skills/.

/cortex-sync
Pull cortex to a new machine or push config changes back.
/cortex-init
Scaffold a new project entry with summary, CLAUDE.md, backlog, and skills directory.
/cortex-discover
Audit a project for missing files, stale content, stuck backlog items.
/cortex-consolidate
Find patterns across all project findings. Surfaces what shows up in more than one place.
bring your own
Drop skills like /humanize, /pipeline, /release in ~/.cortex/global/skills/ and cortex link wires them up. Starter templates have examples.
