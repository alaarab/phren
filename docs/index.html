git-backed memory

Your team memory. Still yours.

cortex keeps long-term engineering memory in files, not a black box. It pulls relevant context into prompts, captures lessons from real work, and keeps policy/governance in plain markdown and JSON.

Install from npm View source
$ npx @alaarab/cortex init
Recommended default: MCP on. Optional: --mcp off for hooks-only fallback.

What You Get

Built for people who live in terminals and pull requests: fast retrieval, strict provenance, and maintenance that does not become a second job.

Automation

Runs in the flow

Context on prompt submit, sync on session start, and save hooks at session end for supported tools.

Control

MCP when you want it

Default to MCP on, or switch to hooks-only mode. Setup covers detected Claude Code, VS Code, Cursor, Copilot CLI, and Codex installs.

Governance

Trustable memory

Citation-aware filtering, retention and decay policy, role controls, and audit logs you can diff.

Quick Start

npx @alaarab/cortex init

# Optional mode at install
npx @alaarab/cortex init --mcp off

# Toggle later
npx @alaarab/cortex mcp-mode status
npx @alaarab/cortex mcp-mode on
npx @alaarab/cortex mcp-mode off

What init actually does

  • creates or updates ~/.cortex
  • writes MCP config for detected tools (if enabled)
  • registers lifecycle hooks for supported local agents
  • applies machine/profile wiring for multi-machine setups
