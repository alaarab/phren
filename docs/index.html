How phren works

Always paying attention. Never in the way.

01
Phren listens and learns

Phren catches findings, links fragments across projects, and figures out what matters. Gets sharper the longer you use it.

02
Follows you everywhere

Switch laptops mid-thought. Already there. Git syncs after every response, pulls on open. Your memory travels with you, not your machine.

03
Peek inside the brain anytime

Runs in the background. When you want to look, phren opens the shell. phren web-ui shows the full memory graph. Always an open book.

Under the hood

Three hooks. Every knob yours.

SessionStart on open
git pull sync from remote
detectProject(cwd) match to phren project
inject(context) instructions enter context window
UserPromptSubmit every message
extractKeywords bigrams, stopword filter
fts5.search synonyms, sub-ms lookup
trustFilter decay, citation, threshold
inject(top3) budget-capped context
Stop after response
git commit save changes
background push rebase on conflict
Configurable
Hooks

Per tool, per project, or off entirely. MCP-only mode available.

Sensitivity

Minimal / conservative / balanced / aggressive.

Token budget

180 for light. 550 default. 2000 for deep context.

Trust + decay

TTL, confidence floor, decay curves. Pin to override.

Sharing + storage

Profiles per machine. Global, project-local, or custom path.

Phren's tricks

55 MCP tools built in.
Skills for the bigger stuff.

MCP tools handle search, capture, review, and sync. Skills are shortcuts for larger jobs. Optional. Drop new ones in ~/.phren/global/skills/.

/phren-init
Point phren at a new project. Sets up files and starts learning.
/phren-consolidate
Tidies up. Finds patterns, deduplicates, promotes what matters.
teach it yours
Drop skills into ~/.phren/global/skills/ and phren picks them up on next init.
