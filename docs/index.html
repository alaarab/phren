Why cortex

Built around how
coding agents actually work.

Context windows are finite and tokens cost money. Cortex makes your agent precise about what it loads, pulling only the relevant slice instead of the whole file.

Search, not load

Claude searches your knowledge base for what it needs instead of loading everything upfront. Synonym expansion finds related terms automatically: search "throttling" and it also finds "rate limit" and "429".

Automatic context

Before each prompt, hooks search your knowledge base and inject matching results: nothing if there's no match, top 3 results (~400 tokens) if there is. Learnings auto-commit when the session ends. Zero manual work.

Get started

One command.

Sets up your knowledge base, configures the MCP server, and registers context hooks for every detected agent (Claude Code, Copilot CLI, Cursor, Codex) automatically.

$ npx @alaarab/cortex init
Creates ~/.cortex with starter templates, registers your machine, configures the MCP server, and wires up hooks for every detected agent. Restart your agent and you're done.
$ npx @alaarab/cortex init
Detects VS Code automatically and writes mcp.json for you. Or add it manually: 
{
  "servers": {
    "cortex": {
      "command": "npx",
      "args": ["@alaarab/cortex", "~/.cortex"]
    }
  }
}
$ npx @alaarab/cortex init
Detects Cursor automatically and writes hooks to ~/.cursor/hooks.json. Prompt injection and auto-save both work. Note: Cursor has no session-start hook, so context re-injection after compaction isn't available.
For MCP, add to your Cursor settings: 
{
  "mcpServers": {
    "cortex": {
      "command": "npx",
      "args": ["@alaarab/cortex", "~/.cortex"]
    }
  }
}
$ npx @alaarab/cortex init
Detects Codex automatically and writes hooks to codex.json in your cortex directory. Per-project CLAUDE.md is also symlinked as AGENTS.md in each project for full instruction file support.
Context injection, auto-save, and session start/end hooks all work with Codex out of the box.
Skills

Full MCP surface. Four skills.
Yours to extend.

The MCP server exposes a full tool surface for search, backlog management, learning capture, and memory governance. These four skills cover what needs a human trigger. Your own go in ~/.cortex/global/skills/.

/cortex-sync
Pull cortex to a new machine or push config changes back. Run this on first setup or after switching machines.
/cortex-init
Scaffold a new project entry with summary, CLAUDE.md, backlog, and skills directory.
/cortex-discover
Research what's missing in a project. Returns a prioritized list of gaps and opportunities.
/cortex-consolidate
Find patterns across all your project LEARNINGS.md files. Surfaces what applies everywhere.
bring your own
Workflow skills like /humanize, /pipeline, /release are personal. Drop them in ~/.cortex/global/skills/ and cortex link wires them up. The starter template (bundled in the package) has examples.
