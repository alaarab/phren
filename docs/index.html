How it works

Every session, automatically.

01
Findings, entities, and memory captured

Claude calls MCP tools as you work. Findings go to FINDINGS.md, entities link across projects, and the confidence system decides what stays relevant. The store gets smarter the longer you use it.

02
Synced across machines automatically

The Stop hook commits locally and queues background sync pushes after every response. SessionStart pulls on open. Switch machines mid-task. Profiles control which projects each machine sees.

03
Shell and UI when you need them

MCP handles the daily flow. When you want to browse or clean up: cortex opens the interactive shell. cortex web-ui opens the browser with your memory graph and review queue.

Under the hood

Three default hooks. Optional PostToolUse.

SessionStart once, on open
git pull --rebase sync ~/.cortex from remote
detectProject(cwd) match working directory to a cortex project
inject(CLAUDE.md, summary.md) project instructions enter the context window
↳ your agent knows the project before you type anything
UserPromptSubmit every message you send
extractKeywords(prompt) strip stopwords, build bigrams
fts5.search(keywords) synonym expansion, 20-term cap, sub-ms lookup
trustFilter(results) confidence decay, citation check, policy threshold
inject(top3, budget=550) best matches enter context. nothing if nothing matches.
↳ ◆ cortex · 3 results in your status line, that's this hook reporting what it found
Stop after every response, including subagents
git add -A ~/.cortex stage whatever changed
git commit no-op if nothing changed
background sync worker push later, rebase on conflict, auto-retry
↳ next session, any machine: SessionStart pulls it back down

Workflow

MCP tools handle the work.
Skills are optional shortcuts.

Cortex exposes MCP tools organized by workflow: search and retrieve context, capture findings and tasks, review memory quality, and sync across machines. Skills are shortcuts for common multi-step jobs. Not required for daily use. Add your own in ~/.cortex/global/skills/.

/cortex-sync
Pull cortex to a new machine or push config changes back.
/cortex-init
Scaffold a brand-new project entry. Use cortex add for an existing repo.
/cortex-discover
Audit a project for missing files, stale content, stuck task items.
/cortex-consolidate
Find patterns across all project findings. Surfaces what shows up in more than one place.
/cortex-profiles
Manage machine-to-profile mappings. Add machines, switch profiles, and keep multi-machine config in sync.
bring your own
Drop your own skills into ~/.cortex/global/skills/ and cortex init wires them up on the next refresh. Starter templates include examples without prescribing your workflow.
