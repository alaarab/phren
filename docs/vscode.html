VS Code Extension

Browse, search, and manage phren knowledge directly from VS Code. Activity bar tree view, fragment graph webview, inline finding capture, and full settings UI.

Overview #

The phren VS Code extension (phren-vscode) provides a native editor experience for your phren knowledge store. It communicates with the phren MCP server over JSON-RPC and gives you:

  • An activity bar panel with a tree view of all projects, findings, tasks, skills, hooks, and sessions
  • Full-text search across all knowledge via a quick pick (Ctrl+Shift+K)
  • An interactive fragment graph webview with Barnes-Hut force simulation
  • Inline finding capture from editor selections via the right-click context menu
  • Task management — add, complete, pin, update, and remove tasks from the tree view
  • Session overview panels showing findings and tasks from past sessions
  • A project config panel for per-project settings
  • Sync, doctor, and hooks status commands accessible from the command palette

Installation #

The extension requires phren to be installed globally first:

npm install -g @phren/cli
phren init

Then install the extension from the VS Code marketplace or build from source:

# From marketplace
ext install alaarab.phren-vscode

# Or build from source
cd ~/phren/vscode-extension
npm install
npm run compile
# Then "Install from VSIX" or run in Extension Development Host
Tip

The extension auto-detects the MCP server from your global npm install. If auto-detection fails, set phren.mcpServerPath in settings to the absolute path to the phren MCP server index.js.

First Run

On first activation, the extension runs an onboarding flow that verifies your phren installation and configures the MCP connection. It checks for:

  • A valid ~/.phren store directory
  • The MCP server entrypoint (auto-detected or configured via phren.mcpServerPath)
  • A working Node.js binary (VS Code's Electron binary won't work — phren.nodePath must point to a real node install)
Important

VS Code's built-in Electron binary cannot run the MCP server. If you see spawn errors, set phren.nodePath to your system Node.js path (e.g. /usr/bin/node or the output of which node).

Tree View #

The phren activity bar icon opens the Explorer tree view. It shows a hierarchical view of your active project:

  • Projects — switch between projects with Set Active Project
  • Findings — browse all findings, filter by date, right-click to remove, supersede, retract, or resolve contradictions
  • Tasks — view active, queued, and done tasks with inline actions (complete, pin, update, remove)
  • Skills — browse installed skills, click to open in editor, toggle enable/disable
  • Hooks — see hook status for each tool (Claude, Copilot, Cursor, Codex), toggle from the tree
  • Sessions — browse session history, open session overview panels, copy session IDs
  • Review Queue — pending review items with inline detail views

The tree view title bar has quick-access buttons for:

  • Search — open the search quick pick
  • Graph — open the fragment graph webview
  • Refresh — reload tree data from the MCP server
  • Sync — commit and push phren changes

Search #

Press Ctrl+Shift+K (or Cmd+Shift+K on macOS) to open the search quick pick. This searches across all indexed content using phren's FTS5 search with synonym expansion.

You can also trigger search from:

  • The command palette: Phren: Search Knowledge
  • The search icon in the tree view title bar

Results show the memory title and a snippet. Select a result to view the full content.

Fragment Graph #

The fragment graph webview renders an interactive force-directed graph of your knowledge fragments and their connections. It uses a Barnes-Hut force simulation with relevance gravity for layout.

Open it from:

  • The command palette: Phren: Show Fragment Graph
  • The graph icon in the tree view title bar

The graph loads the same rendering engine used by phren web-ui, embedded in a VS Code webview panel.

Findings #

Capture findings directly from your editor:

  • From selection — select text in the editor, right-click, and choose Phren: Add Finding. The selected text becomes the finding content.
  • From command palette — run Phren: Add Finding and type the finding text.
  • From tree view — right-click any finding to remove, supersede, retract, or resolve contradictions.

Finding Lifecycle

Right-click a finding in the tree view for lifecycle actions:

  • Remove Finding — delete the finding
  • Supersede Finding — mark as superseded by a newer finding
  • Retract Finding — retract with a reason
  • Resolve Contradiction — resolve conflicts between contradicted findings

Filter by Date

Click the calendar icon on the Findings category to filter findings by date range.

Tasks #

Full task management from the tree view:

  • Add Task — command palette or add icon
  • Complete Task — inline checkmark icon on active tasks
  • Update Task — inline edit icon to change text, priority, or section
  • Pin Task — inline pin icon to keep a task visible across sessions
  • Remove Task — inline trash icon

GitHub Integration

Right-click a task to access GitHub integration:

  • Link GitHub Issue — link an existing GitHub issue to a task
  • Create GitHub Issue — create a new issue from the task and link it back

Sessions #

Browse past sessions in the tree view. Each session shows its timestamp and project. Right-click actions:

  • Open Session Overview — opens a detail panel showing all findings and tasks from that session
  • Copy Session ID — copies the session ID to clipboard for use with CLI or MCP tools

Skills & Hooks #

Manage skills and hooks directly from the tree view:

  • Skills — click a skill to open it in the editor. Use Toggle Skill to enable or disable without deleting.
  • Hooks — see per-tool hook status (Claude Code, Copilot, Cursor, Codex). Click the inline toggle to enable or disable hooks for a specific tool.

Additional commands from the palette:

  • Phren: Hooks Status — show full hook status
  • Phren: Toggle Hooks — global hooks toggle

Profiles & Machines #

Manage multi-machine setups from VS Code:

  • Phren: Set This Machine's Profile — select which profile this machine uses
  • Phren: Set Machine Alias — set a friendly name for this machine
  • Phren: Open machines.yaml — open the machines config file in the editor

Settings #

All settings are in VS Code's Settings UI under "Phren". They are organized into four groups:

Setup

SettingDefaultDescription
phren.mcpServerPathauto-detectAbsolute path to the phren MCP server index.js. Leave blank to auto-detect from a global npm install.
phren.nodePathnodePath to the Node.js binary. Must be a real node install, not VS Code's Electron binary.
phren.storePath~/.phrenPath to the phren store directory. Leave blank for the default.
phren.onboardingCompletefalseInternal flag. Reset to false to re-trigger first-run onboarding.

Capture Behavior

SettingDefaultDescription
phren.findingSensitivitybalancedMaster control for capture aggressiveness. Options: minimal, conservative, balanced, aggressive. Sets session cap and agent instruction level.
phren.proactivityhighGlobal proactivity for findings and tasks. Options: high (always auto-capture), medium (explicit signals only), low (fully manual).
phren.proactivityFindingshighOverride proactivity for findings only. Takes priority over the global setting.
phren.proactivityTaskshighOverride proactivity for tasks only. Takes priority over the global setting.
phren.taskModeautoTask automation mode. Options: off, manual, suggest, auto.
phren.autoExtracttrueExtract insights from project context on each prompt via hook-prompt.
phren.autoCapturefalseExtract insights from conversation transcripts at session end. Requires an active LLM endpoint.

Hooks & Integration

SettingDefaultDescription
phren.hooksEnabledtrueMaster switch for phren hooks. When disabled, phren becomes a passive MCP server.

Semantic Analysis (Offline)

SettingDefaultDescription
phren.semanticDedupfalseSemantic duplicate detection for offline maintenance (phren maintain consolidate).
phren.semanticConflictfalseSemantic conflict detection for offline maintenance.
phren.llmModelserver defaultLLM model for offline operations. Options: claude-haiku-4-5-20251001, claude-sonnet-4-6, gpt-4o-mini, gpt-4o. Haiku recommended for cost.
Note

Semantic dedup and conflict detection use the active agent during live sessions at no extra cost. The llmModel setting only applies to offline batch operations like phren maintain consolidate.
