Getting Started

Install phren, create your first project, and start building persistent memory for your AI agents.

Installation #

Five steps from zero to working memory.

01
Install

phren is published as @phren/cli on npm. Install it globally:

npm install -g @phren/cli
Requires

Node.js 18 or later. Check with node --version.

02
Initialize

Run init to configure hooks and MCP for your agents:

phren init

Expected output:

phren init v0.0.11
  Detected agents: Claude Code, Copilot
  Created ~/.phren/
  Registered MCP server in ~/.claude/settings.json
  Registered hooks: SessionStart, UserPromptSubmit, Stop
  Wrote Copilot MCP config
  Initialized git repo in ~/.phren
  Done. Run 'phren verify' to check setup.

Init is idempotent: re-running it updates MCP config and hooks without clobbering your data. It detects which AI tools you have installed (Claude Code, Copilot, Cursor, Codex) and configures each one.

03
Add a Project

Add your project directory:

# From inside your project directory
phren add

# Or specify a path
phren add ~/my-project

Expected output:

Added project 'my-project' to profile 'main'
  Created ~/.phren/my-project/CLAUDE.md
  Created ~/.phren/my-project/FINDINGS.md
  Created ~/.phren/my-project/tasks.md
  Created ~/.phren/my-project/summary.md

This creates ~/.phren/my-project/ with:

  • CLAUDE.md — project-specific agent instructions
  • FINDINGS.md — captured insights and patterns
  • tasks.md — task queue (active, queued, done)
  • summary.md — project summary for context injection
04
Start Working

Open your agent and start coding. phren works automatically in the background:

  • Every session start pulls the latest state from git
  • Every prompt gets relevant context injected from your knowledge store
  • Every finding and task is persisted back to git after each response
Tip

You can browse your project store anytime with phren shell or phren web-ui.

05
Verify

Confirm everything is wired up:

phren verify

Expected output:

phren verify
  Root: ~/.phren (shared mode)
  Profile: main (3 projects)
  Hooks: Claude Code (SessionStart, UserPromptSubmit, Stop)
  MCP: registered in ~/.claude/settings.json
  Git: clean, remote configured
  All checks passed.

For agent-specific checks:

phren doctor --agents

Expected output:

phren doctor --agents
  Claude Code .... ok (hooks registered, MCP active)
  Copilot ....... ok (MCP config found)
  Cursor ........ not detected
  Codex ......... not detected
  2/4 agents configured, 0 issues found.

Quickstart #

The fastest path from zero to working memory — combines all five steps above:

phren quickstart

This runs phren init followed by an interactive project scaffolding wizard. It creates the ~/.phren directory, sets up git tracking, and registers your first project.

Your First Project #

After init, add additional project directories at any time:

phren add ~/frontend-app
phren add ~/backend-api

Each project gets its own findings, tasks, and configuration. See Adding Projects for ownership modes.

Templates #

Init supports project templates that scaffold starter files tailored to your project type:

phren init --template python-project
TemplateWhat it scaffoldsBest for
python-project CLAUDE.md with Python conventions, venv/pip patterns, pytest findings Single Python packages, CLI tools, data pipelines
monorepo CLAUDE.md with workspace-aware config, cross-package patterns, CI findings Multi-package repos (npm workspaces, Turborepo, Lerna)
library CLAUDE.md with API surface tracking, semver patterns, consumer impact findings Published libraries and SDKs
frontend CLAUDE.md with component patterns, build tooling, browser compat findings React, Vue, Angular, or vanilla frontend apps
Note

Templates only affect the initial CLAUDE.md and summary.md content. You can change anything after creation. The default template (no flag) works for any project type.

How It Works #

phren operates on three paths during every prompt cycle:

  1. Retrieval — keyword extraction from your prompt, synonym expansion, FTS5 search, and snippet injection into agent context.
  2. Governance — citation validity checks, confidence decay, policy thresholds to filter low-quality memories.
  3. Persistence — MCP tools write findings/tasks, the Stop hook auto-commits and pushes to git.

Hooks run at lifecycle points:

  • SessionStart — git pull on ~/.phren, project context injection
  • UserPromptSubmit — search + inject relevant context on every prompt
  • Stop — auto-commit and push after every response

Here is what context injection looks like in practice:

Adding Projects #

Add as many projects as you want. Each gets its own findings, tasks, and config:

phren add ~/frontend-app
phren add ~/backend-api
phren add ~/shared-lib

Projects support ownership modes:

  • owned — full read/write, hooks active (default)
  • observed — read-only context injection, no writes
  • archived — excluded from active profile, data preserved
phren add ~/legacy-app --ownership observed
phren projects configure legacy-app --ownership=archived

Profiles & Machines #

Profiles map sets of projects to machine roles. Useful when you work on different projects on different machines.

# ~/.phren/profiles/work.yaml
projects:
  - frontend-app
  - backend-api
  - shared-lib

# ~/.phren/profiles/personal.yaml
projects:
  - side-project
  - dotfiles

Map machines to profiles in ~/.phren/machines.yaml:

machines:
  work-laptop: work
  home-desktop: personal
  server: work

List machines and profiles:

phren config machines
phren config profiles

Syncing Across Machines #

~/.phren is a git repo. Sync it across machines by pushing/pulling:

# On your current machine
phren push_changes "sync from laptop"

# On a new machine
phren init  # pulls from remote

Or use the /phren-sync skill for a guided sync experience. The SessionStart hook automatically runs git pull at the start of every session, so you always have the latest data.

Claude Code #

phren init automatically configures Claude Code. It registers:

  • MCP server in ~/.claude/settings.json
  • Hooks for SessionStart, UserPromptSubmit, and Stop events

After init, phren works automatically. Every prompt gets relevant context injected, and every finding is persisted to git.

Copilot #

phren detects GitHub Copilot during init and configures the MCP server. Copilot accesses phren via MCP tools in chat mode.

Cursor #

Cursor is configured via its MCP settings. phren init writes the config to the correct location. Use MCP tools in Cursor's agent/composer mode.

Codex #

OpenAI Codex CLI is supported via MCP. phren init detects it and registers the server.

For all agents, verify the setup with:

phren doctor --agents

Troubleshooting #

Node.js Version Mismatch

phren requires Node.js 18+. If you see syntax errors or ERR_UNSUPPORTED_ESM_SYNTAX:

node --version   # must be v18.0.0 or later
nvm use 18       # if using nvm

Permission Errors

If npm install -g fails with EACCES, fix npm's global prefix:

mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
# Add to ~/.bashrc or ~/.zshrc:
export PATH="$HOME/.npm-global/bin:$PATH"
Warning

Do not use sudo npm install -g. This creates root-owned files in ~/.phren that cause permission issues later.

Corporate Proxy / TLS Issues

If you are behind a corporate proxy (ZScaler, Netskope, etc.) and see UNABLE_TO_GET_ISSUER_CERT_LOCALLY:

# Point Node.js at your corporate CA bundle
NODE_EXTRA_CA_CERTS="/path/to/corporate-ca.crt" phren init

# Common locations:
# macOS: /Library/Application Support/Netskope/STAgent/data/nscacert.pem
# Linux: /usr/local/share/ca-certificates/zscaler.crt
# Windows: C:\ProgramData\ZScaler\ZScalerRootCA.pem
Important

Do not set NODE_TLS_REJECT_UNAUTHORIZED=0. This disables all TLS verification and is a security risk. Use NODE_EXTRA_CA_CERTS instead.

MCP Connection Failures

If your agent cannot connect to the phren MCP server:

  • Run phren doctor --agents to check which agents are configured
  • Verify the MCP config path: ~/.claude/settings.json for Claude Code, check phren hooks list for others
  • Ensure the node path in the MCP config resolves correctly (use absolute paths, not npx)
  • Restart the agent after making config changes
# Quick fix: re-run init to regenerate MCP config
phren init

# Or run doctor with auto-fix
phren doctor --fix

Hooks Not Firing

If context is not being injected or changes are not being committed:

  • Check hook status: phren hooks list
  • Check for recent errors: phren hooks errors
  • Ensure hooks mode is enabled: phren hooks-mode on
  • Re-register hooks: phren init (idempotent, safe to re-run)
# View hook status
phren hooks list

# Check for errors
phren hooks errors --limit 5

Expected healthy output:

phren hooks list
  Claude Code:
    SessionStart .... enabled (hook-session-start)
    UserPromptSubmit  enabled (hook-prompt)
    Stop ............ enabled (hook-stop)
  Custom hooks: none
  Recent errors: 0
