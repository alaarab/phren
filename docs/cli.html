CLI Reference

All phren CLI commands, organized by category. Run phren --help for a quick summary.

General #

phren
Launch the interactive shell. Browse findings, run searches, manage tasks interactively.
phren quickstart
Quick setup: runs init + scaffolds your first project in one step.
phren status
Health, project, and stats overview. Shows active profile, project count, finding counts.
$ phren status
Profile:   default (3 projects)
Machine:   dev-laptop
Health:    OK (all checks passed)
Findings:  142 active, 23 archived
Tasks:     8 active, 4 queued
Sessions:  Last: 2h ago (my-app)
Index:     1,247 docs, last built 4m ago
phren doctor [--fix] [--agents]
Health check and self-heal. Validates hooks, MCP config, project integrity. Use --fix to auto-repair issues, --agents to check agent tool configs.
FlagDescription
--fixAuto-repair detected issues (re-link hooks, fix symlinks)
--agentsAlso check agent tool configs (Claude, Copilot, Cursor, Codex)
 
$ phren doctor
[OK] ~/.phren exists
[OK] Profile "default" loaded (3 projects)
[OK] Claude hooks registered
[OK] MCP server configured
[WARN] Copilot hooks not detected
[OK] FTS5 index healthy (1,247 docs)

$ phren doctor --fix --agents
[FIX] Re-registered Copilot hooks
[OK] All 4 agent configs verified
phren verify
Post-init verification checks. Validates that hooks, MCP server, and project wiring are all correctly configured.
phren web-ui [--port=3499]
Launch the memory web UI. Browse findings, tasks, and the knowledge graph in your browser.
phren update [--refresh-starter]
Update to the latest version. Use --refresh-starter to also update bundled starter templates.
phren uninstall
Remove phren config and hooks. Data in ~/.phren is preserved unless manually deleted.
phren mcp-mode [on|off|status]
Toggle MCP integration on or off, or check current status.
phren hooks-mode [on|off|status]
Toggle hook execution on or off, or check current status.

Search #

phren search <query> [--project <n>]
Search the knowledge base using FTS5 with synonym expansion. Optionally scope to a specific project.
FlagDescription
--project <name>Filter results to a single project
--historyShow recent search history
--from-history <n>Re-run search entry #n from history
 
$ phren search "authentication middleware"
  [my-app] [pitfall] Auth middleware runs before rate limiting (0.92)
  [backend] [decision] Use passport.js for all auth flows (0.87)
  [my-app] [pattern] Middleware stack order matters for error handling (0.74)

$ phren search "redis cache" --project backend
  [backend] [architecture] Redis used as session store and rate limiter (0.95)
  [backend] [pitfall] Redis connection pool exhaustion under load (0.81)
phren search --history
Show recent search history.
$ phren search --history
  1. "authentication middleware" (3 results, 2m ago)
  2. "redis cache" (2 results, 15m ago)
  3. "deployment pipeline" (5 results, 1h ago)

$ phren search --from-history 1  # re-run search #1
phren search --from-history <n>
Re-run search #n from the search history.
phren graph [--project <n>]
Show the fragment knowledge graph. Displays connections between findings, files, and concepts.
phren graph link <project> "finding" "fragment"
Manually link a finding to a fragment in the knowledge graph.

Tasks #

phren tasks
Cross-project task view. Shows all active and queued tasks across all projects.
$ phren tasks
my-app
  Active:
    A1. [high] Fix auth token refresh race condition
    A2. [medium] Add rate limiting to /api/upload
  Queue:
    Q1. Refactor middleware stack ordering
    Q2. Add integration tests for OAuth flow

backend
  Active:
    A1. [high] Migrate Redis from v6 to v7
phren task add <project> "..."
Add a task to a project's queue.
phren task add my-app "Add input validation to all API endpoints"
phren task complete <project> "..."
Complete a task by matching its text.
phren task complete my-app "Fix auth token refresh"
phren task remove <project> "..."
Remove a task by matching text.
phren task update <project> "..." [--priority=high|medium|low] [--section=Active|Queue|Done]
Update a task's priority or move it to a different section.
phren task next [project]
Pick the next highest-priority task to work on.
phren task promote <project> "..." [--active]
Promote a task to a higher priority section.
phren task pin <project> "..."
Pin a task so it stays visible across sessions.
phren task tidy [project] [--keep=<n>]
Archive completed tasks to keep the list clean. Use --keep to retain the most recent N done items.
phren task link <project> "..." --issue <n>
Link a GitHub issue to a task.
phren task create-issue <project> "..." [--repo <owner/name>] [--done]
Create a GitHub issue from a task item and link it back.
phren task reorder <project> "..." --rank=<n>
Reorder a task within its section by rank number.

Findings #

phren add-finding <project> "..."
Save an insight to a project's FINDINGS.md. Auto-tags if no type prefix is provided.
phren add-finding my-app "[pitfall] Auth middleware runs before rate limiting — always check order"
phren add-finding backend "[decision] Use Zod for all API input validation — better TS inference than Yup"
phren add-finding my-app "Always call db.close() in finally blocks"  # auto-tagged as [pattern]
phren pin <project> "..."
Pin canonical memory to truths.md. Pinned memories are always injected and never decay or get pruned.
phren pin my-app "Deploy to staging first, then prod. Never skip staging"
phren finding add <project> "..."
Add a finding (same as add-finding).
phren finding remove <project> "..."
Remove a finding by matching its text.
phren finding supersede <project> "..." --by "..."
Mark a finding as superseded by a newer one. Superseded findings score 0.25x in retrieval.
phren finding supersede my-app "Use Express for routing" --by "Migrated to Fastify for better perf"
phren finding retract <project> "..." --reason "..."
Retract a finding with a lifecycle reason. Retracted findings score 0.1x in retrieval.
phren finding retract my-app "Redis is single-threaded" --reason "Incorrect — Redis 7 has I/O threading"
phren finding contradictions [project]
List all unresolved contradicted findings.
phren finding contradictions my-app
phren finding resolve <project> "..." "..." <keep_a|keep_b|keep_both|retract_both>
Resolve a contradiction between two findings.
phren finding resolve my-app "Always use JWT" "Session cookies are safer" keep_b
phren sessions [session-id]
Session history. Drill into a specific session to see its findings and tasks.
phren review [project]
Show review queue items waiting for approval.
phren consolidation-status [project]
Check if findings need consolidation.
phren session-context
Show current session state: project, duration, findings added so far.

Projects #

phren add [path] [--ownership <mode>]
Add current or specified directory as a project.
FlagDefaultDescription
--ownershipphren-managedHow phren treats instruction files. Options: phren-managed, detached, repo-managed
 
phren add                                    # add current directory
phren add /path/to/repo --ownership=repo-managed
phren projects list
List all projects in the active profile.
phren projects list
phren projects configure my-app --ownership=repo-managed
phren projects remove old-project
phren projects configure <name> --ownership=<mode>
Change a project's ownership mode (phren-managed, detached, repo-managed).
phren projects remove <name>
Remove a project (confirmation required). Data is preserved in ~/.phren.

Skills #

phren skills list
List all installed skills with metadata.
$ phren skills list
Global skills:
  phren-sync        Pull/push ~/.phren across machines
  phren-init        Scaffold a new project
  phren-discover    Research gaps and opportunities

my-app skills:
  deploy-checklist  Pre-deploy verification steps
phren skill-list
Alias for skills list.
phren detect-skills [--import]
Find untracked skills in ~/.claude/skills/. Use --import to bring them into phren.
phren detect-skills           # scan and report
phren detect-skills --import  # scan and import found skills
phren skills add <project> <path>
Add a skill to a project.
phren skills show <name> [--project <name>]
Show the full content of a skill.
phren skills edit <name> [--project <name>]
Edit a skill in $EDITOR.
phren skills resolve <project|global>
Print the resolved skill manifest (effective skills after inheritance).
phren skills doctor <project|global>
Diagnose skill visibility issues.
phren skills sync <project|global>
Regenerate skill mirror files.
phren skills enable <project|global> <name>
Enable a disabled skill without rewriting it.
phren skills disable <project|global> <name>
Disable a skill without deleting it.
phren skills remove <project> <name>
Remove a skill from a project.

Hooks #

phren hooks list [--project <name>]
Show hook enable/disable status per tool, with optional project overrides.
phren hooks list
phren hooks list --project my-app
phren hooks show <tool>
Show hook config for a specific tool (claude, copilot, cursor, codex).
phren hooks edit <tool>
Edit hook config in $EDITOR.
phren hooks enable <tool>
Enable hooks for a tool.
phren hooks enable claude
phren hooks enable cursor
phren hooks disable <tool>
Disable hooks for a tool.
phren hooks disable codex
phren hooks add-custom <event> <cmd>
Add a custom integration hook for a lifecycle event.
phren hooks add-custom Stop "curl -s http://localhost:9090/notify"
phren hooks remove-custom <event>
Remove custom hooks by event match.
phren hooks errors [--limit <n>]
Show recent hook errors and failures.

Config #

Tip

All config commands support get (read current value) and set (update). Run with no args to see current values.

phren config policy [get|set ...]
Manage retention policy: TTL, confidence, decay settings.
phren config policy get
phren config policy set memoryTTLDays=90 decayMultiplier=0.8
phren config workflow [get|set ...]
Manage workflow: approval gates, risky thresholds.
phren config workflow set taskMode=auto
phren config workflow set findingSensitivity=high
phren config access [get|set ...]
Role-based permissions configuration.
phren config index [get|set ...]
Indexer include/exclude globs for FTS5.
phren config synonyms [list|add|remove] ...
Manage learned synonyms for search expansion.
phren config synonyms list
phren config synonyms add "auth" "authentication,authorization"
phren config synonyms remove "auth"
phren config project-ownership [mode]
Set the default ownership mode for new projects.
phren config telemetry [on|off]
Toggle opt-in usage telemetry.
phren config machines
List registered machines.
phren config profiles
List profiles and their associated projects.

Maintenance #

Warning

Run prune and consolidate with --dry-run first. Phren creates FINDINGS.md.bak automatically on write paths that modify the file.

phren maintain govern [project]
Queue stale memories for review based on retention policy.
phren maintain govern my-app
phren maintain prune [project]
Delete expired entries that have passed their TTL.
phren maintain prune              # all projects
phren maintain prune my-app       # single project
phren maintain consolidate [project]
Deduplicate FINDINGS.md: archive old entries, write consolidation markers, promote cross-project patterns.
phren maintain consolidate my-app
phren maintain consolidate my-app --dry-run  # preview without writing
phren maintain extract [project]
Mine git and GitHub signals (PRs, CI, issues) to extract memories automatically.
phren maintain extract backend
