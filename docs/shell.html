Interactive Shell

A terminal UI for browsing and managing your entire project store. Launch it with phren or phren shell.

Launching #

The shell starts automatically when you run phren with no arguments in a terminal:

phren

Or explicitly:

phren shell

Views #

The shell has eight views. Switch between them with single-key shortcuts or the : palette.

Projects p

The landing screen. Opens as a dashboard showing profile-wide counts for projects, active tasks, queued tasks, findings, review queue items, sync state, and intro mode.

Below the dashboard, all indexed projects in the active profile are listed with per-project workload counts. The currently selected project is marked with a green dot.

  • Press Enter or use :open <project> to set the active project context
  • The active project persists across all views
  • Press i to cycle intro mode on the dashboard

Tasks b

Shows the selected project's tasks.md parsed into three sections: Active, Queue, and Done. Each item has an auto-generated ID for easy reference in commands.

  • Cross-session checkpoint context is reflected through task metadata and lifecycle actions
  • Use :work next to promote the top Queue item to Active
  • Use :complete <id> to mark items done
  • Use :tidy to archive old Done items

Findings l

Lists entries from the selected project's FINDINGS.md with dates and optional citations. Entries are shown with auto-generated IDs for use with :find remove.

  • Finding lifecycle states (superseded, retracted, contradicted) are visible here
  • Provenance markers help with cleanup and contradiction resolution

Review Queue m

Shows pending review items from review.md. Each item displays its section (Review, Stale, Conflicts), a risk badge, confidence score, and date.

  • The shell review queue is read-only and meant for inspection
  • Use phren maintain prune or update findings directly for mutations
  • The browser view (phren web-ui) is also read-only for queue items

Skills s

Shows installed global and project-local skills in the current context.

  • Press t to toggle enabled/disabled state
  • Skills have three lifecycle states: enabled (active), disabled (excluded from use), and removed (deleted)
  • Project-local scope overrides global scope
  • Alias/command collisions are flagged and not registered

Hooks k

Shows lifecycle hook status and custom hook configuration for the active setup.

  • Claude: native SessionStart / UserPromptSubmit / Stop hooks
  • Copilot/Cursor/Codex: generated hook config + session wrappers

Machines / Profiles :machines

Shows the machines.yaml mapping (hostname to profile) and all profiles with their project lists. Accessed via palette command rather than a single-key shortcut.

  • Use :machine map <hostname> <profile> to assign machines to profiles
  • Use :profile add-project / :profile remove-project to manage projects in profiles
  • Safer than editing YAML directly

Health h

Runs doctor checks and shows results alongside runtime health data: last hook run, last auto-save, last governance run.

  • Offers remediation commands: :run fix, :relink, :rerun hooks, :update
  • Reflects RBAC identity context (PHREN_ACTOR and access-control files)
  • Shows review queue pressure and telemetry status

Keybindings #

Single-key shortcuts for fast navigation. No modifier keys needed.

View Switching

p Projects view
b Tasks view
l Findings view
m Review Queue view
s Skills view
k Hooks view
h Health view
q Quit the shell

In-View Actions

Enter Activate / open selected item
i Cycle intro mode (Projects)
t Toggle enabled/disabled (Skills)
/ Set or clear filter
: Enter palette command

Palette Commands #

All palette commands start with :. You can also type them without the colon prefix.

Project Context

CommandDescription
:open <project>Set the active project context across all views
:intro always|once-per-version|offTune the startup intro behavior

Tasks

CommandDescription
:add <task>Add a new item to the task Queue section
:complete <id|match>Mark a task item as done
:move <id|match> <active|queue|done>Move a task item between sections
:reprioritize <id|match> <high|medium|low>Set an item's priority
:context <id|match> <text>Append or update context on a task item
:work nextMove the top Queue item to Active
:tidy [keep]Archive old Done items (default: keep last 30)

Findings

CommandDescription
:find add <text>Append a finding to the selected project
:find remove <id|match>Remove a finding by ID or text match

Infrastructure

CommandDescription
:machine map <hostname> <profile>Map a machine hostname to a profile
:profile add-project <profile> <project>Add a project to a profile
:profile remove-project <profile> <project>Remove a project from a profile
:run fixRun doctor --fix to self-heal configuration
:relinkRefresh generated links, context files, and hook wiring
:rerun hooksExecute lifecycle hooks now (session-start + stop)
:updateUpdate phren to the latest version
:resetReset shell state to defaults

Pagination

CommandDescription
:page nextGo to the next page
:page prevGo to the previous page
:page <n>Jump to page number n
:per-page <n>Set rows per page (1 to 200)
:helpShow the full keyboard map and command list

Filtering #

Type / followed by text to filter the current view. The filter applies as a case-insensitive substring match against item content.

Type / alone to clear the filter. Filters reset the page to 1.

State Persistence #

The shell saves its state to .runtime/shell-state.json. This includes:

  • Selected project
  • Active filter
  • Current page and rows per page
  • Intro mode and last-seen intro version

Your context and preferences are preserved between shell sessions. Use :reset to clear saved state.
