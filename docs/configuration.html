Environment Variables #

Set these in your shell profile, ~/.phren/.env, or per-project .env files. All are optional.

VariableDefaultDescription
PHREN_FEATURE_GIT_CONTEXT_FILTERdisabledSet to 'true' to enable git-context file relevance filtering in search ranking. Docs matching currently-changed files or branch name get a boost.
PHREN_FEATURE_AUTO_CAPTUREdisabledSet to '1' to extract insights from conversation transcripts at session end (written by init walkthrough to ~/.phren/.env).
PHREN_EMBEDDING_API_URL—OpenAI-compatible /embeddings endpoint (e.g. https://api.openai.com/v1). Takes priority over Ollama when set.
PHREN_EMBEDDING_API_KEY—Bearer token for PHREN_EMBEDDING_API_URL.
PHREN_FEATURE_SEMANTIC_DEDUPdisabledSet to '1' to enable LLM-based semantic deduplication on add_finding. Checks whether a new finding is semantically equivalent to existing ones. Requires LLM configured via PHREN_LLM_ENDPOINT or ANTHROPIC_API_KEY/OPENAI_API_KEY.
PHREN_FEATURE_SEMANTIC_CONFLICTdisabledSet to '1' to enable LLM-based conflict detection on add_finding. Detects contradictions between findings. Appends <!-- conflicts_with: "..." --> annotations.
PHREN_LLM_MODELgpt-4o-mini / claude-haiku-4-5-20251001Override the LLM model used for semantic dedup and conflict detection.
PHREN_LLM_ENDPOINT—OpenAI-compatible /chat/completions base URL for semantic dedup/conflict. Takes priority over Anthropic fallback.
PHREN_LLM_KEY—API key for PHREN_LLM_ENDPOINT. Falls back to OPENAI_API_KEY or ANTHROPIC_API_KEY.

Retention Policy #

Configure via phren config policy or the set_retention_policy MCP tool. Settings are stored in ~/.phren/.governance/retention.json.

SettingDescription
ttl_daysDefault time-to-live in days for findings before they become candidates for pruning.
confidence_decayRate at which finding confidence scores decay over time (0–1 scale).
min_confidenceMinimum confidence threshold. Findings below this are candidates for review/pruning.
max_entriesHard cap on total entries per project before forced consolidation.
auto_accept_thresholdScore threshold (0–1) above which extracted memories skip the review queue.

Workflow Policy #

Configure via phren config workflow or the set_workflow_policy MCP tool. Stored in ~/.phren/.governance/workflow.json.

SettingDescription
approval_requiredWhether findings need manual approval before being committed. Boolean.
risky_thresholdConfidence score below which findings are flagged for review rather than auto-accepted.
auto_consolidateWhether consolidation runs automatically when thresholds are exceeded. Boolean.
push_on_stopWhether the Stop hook auto-pushes changes. Boolean (default: true).

Index Policy #

Configure via phren config index or the set_index_policy MCP tool. Controls which files the FTS5 indexer includes.

SettingDescription
includeArray of glob patterns for files to index (e.g. ["**/*.md", "**/*.txt"]).
excludeArray of glob patterns to skip (e.g. ["**/node_modules/**"]).
max_file_sizeMaximum file size in bytes to index. Files larger than this are skipped.

Proactivity Levels #

Set via set_proactivity(level) MCP tool. Controls how aggressively phren injects context and captures findings.

LevelBehavior
offNo automatic context injection or finding capture. Manual only via MCP tools.
lowMinimal injection: only high-confidence, recently-accessed findings. No auto-capture.
mediumBalanced: injects relevant context on each prompt, captures findings when confidence is high. Default level.
highAggressive: broader context injection, auto-capture enabled, consolidation notices, cross-project context included.

Finding Sensitivity #

Set via set_finding_sensitivity(level) MCP tool. Controls how easily new findings are accepted.

LevelBehavior
lowOnly high-signal findings are captured. Duplicate and low-value detection is strict.
mediumDefault. Standard dedup checks, moderate low-value filtering.
highPermissive: more findings captured, weaker dedup. Good for exploration phases.

Task Mode #

Set via set_task_mode(mode) MCP tool. Controls how tasks are managed and surfaced.

ModeBehavior
manualTasks only added/completed via explicit MCP tool calls or CLI commands.
suggestAgent suggests tasks based on conversation context. User confirms before adding.
autoAgent automatically adds and manages tasks based on conversation flow.

Per-Project Overrides #

Each project in ~/.phren/<project>/ can override global settings. Place a config.json in the project directory with any of the governance settings above. Project-level settings take priority over global ones.

{
  "retention": { "ttl_days": 90 },
  "workflow": { "approval_required": true },
  "proactivity": "low",
  "finding_sensitivity": "high",
  "task_mode": "manual"
}

Projects also support ownership modes via phren projects configure <name> --ownership=<mode>:

  • owned — full read/write access, hooks active
  • observed — read-only context injection, no writes
  • archived — excluded from active profile, data preserved

Hook Configuration #

Hooks run at specific lifecycle points. Manage with phren hooks CLI or list_hooks/toggle_hooks MCP tools.

HookEventWhat it does
SessionStartSession beginsRuns git pull on ~/.phren, executes hook-context for project context injection.
UserPromptSubmitEvery promptExtracts keywords from the prompt, searches phren via FTS5, injects relevant context. Checks consolidation threshold and injects phren-notice once per session.
StopAfter every responseAuto-commits and pushes ~/.phren changes. Fires after subagents too, so consolidation never runs from Stop.

Custom hooks can be added with phren hooks add-custom <event> <command> or the add_custom_hook MCP tool. Hooks can be toggled per-tool (Claude, Copilot, Cursor, Codex) and per-project.
